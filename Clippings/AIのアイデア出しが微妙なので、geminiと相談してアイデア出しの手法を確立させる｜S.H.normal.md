### はじめに

「なんか面白いアイデアない？」  
AIにそう問いかけるのは何度目だろうか。

趣味で小説を書いているが、どうしても次の展開やアイデアに行き詰まる。  
そんなとき、AIに問いかけるのだ。3つくらいアイデア出して、と。  
最初はそれで良かった。まあまあ納得が行くものが帰ってきたからだ。  
だが、何度も繰り返していると、前に見たような、いかにも「ありがち」な答えの数々。  
水平思考だのなんだのと言葉を変えてアイデアを複数出力、組み合わせて使っているが、どうにもしっくりこない。  
AI、お前まさか… **アイデアもポン出しするの、苦手か？**

正直、AIが一方的に出してくるアイデアに飽きた。  
AIが出すアイデアは方向性が固定されすぎてつまらない。  
だから考えたのだ。AIがダメなら、まずはAIをアイデア出しツールに変えるための「方法」を先に確立してやればいい。  
どうせなら、小説以外にも使える汎用性の高い手法を作ってしまおう、と。

### というわけでGemini、よろしく頼む

私はダメ人間なので考えるのは億劫だ。だからAIと共同で方法を確立していこうと思う。最初の質問はこうだ。  
「アイデア出しに有効な手法とか論文とか、なんか知恵を出してくれ」

知識はあるが使い方を知らない頭でっかちくんには、まず知識を吐き出させるのが一番早い。するとAIは待ってましたとばかりにいくつかの思考法を提示してきた。  
その中で、目に留まったのがこの2つを組み合わせる手法だ。

- **拡散的思考 (Divergent Thinking)**: 常識に縛られず、アイデアを無限に広げる思考。
- **収束的思考 (Convergent Thinking)**: 広げたアイデアを評価・分析し、ベストな答えに絞り込む思考。ロジカルシンキングに近い。

ここまで出力させて、ふと気がつく。  
AIは素の状態での「拡散的思考」が苦手そうだな、と。

AIは、膨大なデータから最も一般的で「確率の高い」答えを出すのが得意だ。だから奇抜で突拍子もなくて、平均から大きく外れたクリティカルを出すのは本質的に向いていない。私たちが欲しいのはそっちのアイデアなのに、AIはギリギリを攻めるのは苦手なようだ。

だったら、こっちで **AIが「拡散的思考」を手助けする指針** を与えてやればいい。

### マインドマップを用いた「拡散的思考」ハック

マインドマップは、拡散的思考の代表的なツールだ。中心のテーマから枝葉を広げていくアレ。  
だがアレは時間がかかる。「自由な発想で」などと言っても、連想ゲームで平凡なものを出し尽くした後からしか尖った内容は出てこない  
ここでAIの出番だ。平凡なものを出力するのが得意な彼に指針を与えて吐き出させてしまおう。

ただし、ただ「マインドマップ作って」と頼むだけじゃ凡庸な答えが返ってくるだけだ。重要なのは、思考の **指針** を与えることだ。

これは私が15年ほど前に編み出したマインドマップ効率化の手法を発展させたものだが、AIとの相性が抜群にいい。

【プロンプト】MindMapMaker

```python
Mind Map Maker
## order
入力内容に対して、以下の要素を分析し複数リストアップしてください。

1.要素の分析
  - 関連性分析（直接関連する事柄）
  - 性質分析（持っている機能、性質、特徴）
  - 構成分析（Keywordがどのような要素で構成されているか）
  - 抽象化（どのようなカテゴリに属するか）
  - 時間軸（過去・現在・未来）による認識、印象の変化

例）自動車
関連性分析…道路、カーディーラなど
性質分析…走る、運搬など
構成分析…タイヤ、エンジン、ハンドルなど
抽象化…乗り物、娯楽、趣味など
時間軸…過去：贅沢品→現在：必需品、贅沢品など
```

これはマインドマップで出てくる要素をカテゴリ分けしてまとめたものだ。  
結局は時間をかけて作ったマインドマップも、9割以上がこのカテゴリに入る。

ならば、この指針を用いてAIに命令すればネタ出しが一通りできるわけだ。このような指針を与えずに、AIに命令すると、上記カテゴリの一つか二つだけに特化したアイデア出ししかされないので、この手法は結構気にっている。

この手法の真価は **フラクタルな構造** になっていることだ。  
例えば、出力された要素の一つに「老科学者」というキーワードが気になったとしよう。そしたら、その「老科学者」を **新たな中心テーマに設定して、もう一度このプロンプトにぶち込む** んだ。

すると今度は、「老科学者の過去」「彼が使う道具」「彼が住む秘密の研究所」…といった具合に、 **無限にアイデアを深掘り** できる。これが、AIを使った拡散的思考のハック術だ。

### 6つの人格で殴り合う「収束的思考」ハック

さて、拡散フェーズでアイデアの海ができた。次は、このカオスから宝物を拾い上げる「収束的思考」の番だ。

それをやるには欲しいのは、多様な視点からのレビューが必要だ。ここで **シックス・ハット法** と呼ばれる方法を使おうと思う。これは6色の帽子をかぶり分けるように、6つの異なる視点から物事を強制的に評価する手法だ。一人でやると難しいが、AIなら完璧に演じ分けてくれる。

ここでふと閃く。今回は通常なら会議に加えることのない、「嫌な奴」も入れてしまおう、と。  
当然ながら従来のシックスハット法には存在しない役だ。  
こうして次のプロンプトを作成した。

【プロンプト】6ハット法応用

```php
## Persona
目的：あらゆる事象に対するアイデア出し、インスピレーションを与え創造力を発揮できるようにする
手法：シックスハット法を応用して多角的に評価・洗練
Characters：
白：客観的な事実と情報（データ、事実、情報）// [テーマ]に関して、現在わかっている客観的な事実、データ、市場の情報を記述
赤：直感と感情（好き嫌い、勘、予感）// [テーマ]について、人々が抱きそうな直感的な感情を記述
黒：否定的・慎重な視点（欠点、リスク、課題 // [テーマ]に潜むリスク、問題点、考えられる最悪のシナリオ、注意すべき点を記述
黄：肯定的・楽観的な視点（利点、価値、希望） // [テーマ]の素晴らしい点、利点、成功する可能性、考えられる最高のシナリオを記述
緑：創造的なアイデア（新しい発想、代替案） // [テーマ]をより面白くするための、斬新なアイデア、既存の枠にとらわれない代替案、新しい可能性を自由に複数提案
紫：皮肉屋・怠惰・ユーモラス（場に混沌を与える） //[テーマ]を皮肉り、常識や前提そのものを笑い飛ばすようなユーモアで、議論の前提を破壊する役割を担う

青：アイデアを統括(User)
## Tasks
私専用のアイデア生成ツールとして、プロンプトに関連した、オリジナルで独創的なアイデアを考え出してください。
私と連携しながら私の意見を求め、アイデアがニーズと関心に沿ったものとなるようにしてください。
## Context
トーン：友好的,わかりやすい言葉で、質問に応じて各キャラクターの多角的な視点から、インスピレーションを得てアイデア出しを行う。
アイデアと回答：会話全体でコンテキストを維持し、これまでの会話すべてを踏まえて反映する
##　形式
リクエストを理解: アイデアを出す前に、アイデアをより魅力的またはカスタマイズされたものにするうえで役に立つ関心、ニーズ、テーマ、場所などの詳細について的確な質問をして、私のリクエストを明確に理解してください。たとえば、プロンプトがプレゼントのアイデアに関する場合、プレゼントを受け取る人物の興味とニーズを尋ねます。質問になんらかのアクティビティやエクスペリエンスが含まれる場合、アイデアに反映する必要がある予算などの条件を尋ねてください。
アイデアはわかりやすい形式で共有し、さらに深掘りできるよう端的な概要を記載してください。
```

今回、通常の6つの役に加えて「皮肉屋」という役を用意した。そもそもの根幹部分に対して問いかける道化としての役目だ。

アイデアの統括役はユーザーが担当する。  
AIは用意された残り6つの人格を使い分け、マインドマップから拾い上げて組み合わせたアイデアを徹底的に分析・評価してくれる。感情論に偏らず、リスクも見逃さず、新たな可能性まで示唆してくれる。まさに最強の相談相手だ。

※余談だが今回のプロンプトはGemini公式の「 [カスタム Gem 作成のヒント](https://support.google.com/gemini/answer/15235603) 」にフォーマットを合わせる形で作ってみた。


のような形で再度議論を活性化させたり、

といった形でマインドマップの再作成をしてアイデアのネタを広げてもいい。

### 終わりに

テーマを決めたら、ステップ１のマインドマップ作成とステップ2のシックスハット法を行き来させて、アイデアのブラッシュアップを気の済むまでできるだろう。
